# PAI Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) for the PAI system.

## Purpose

ADRs document significant architectural choices: what changed, why alternatives were rejected, and what evidence drove the decision. They ship with secunit so anyone cloning the public release can understand not just what the system does, but why it is the way it is.

## Format

Each ADR is a Markdown file with this structure:

```markdown
---
name: kebab-case-slug            # used as idempotency key
title: "Human-readable title"
date: YYYY-MM-DD
status: stub | complete
detected: new-subsystem | new-pipeline-domain | algorithm-version | manual
change: "One-line description of what changed"
---

## Decision

What was decided and why.

## Alternatives Rejected

What else was considered and why it was not chosen.

## Evidence

What data, experiments, or reasoning drove this decision.

## Consequences

What follow-on effects or trade-offs result from this decision.
```

## Status Values

- `stub` — auto-generated by ArchitectureSummaryGenerator.ts when a threshold was crossed. **Blocks release until filled.**
- `complete` — human-filled reasoning. Ships with secunit.

## Stub Gate

`release.ts` scans this directory before pushing. Any file with `status: stub` blocks the push and prints the filename. Fill the reasoning sections and change `status: complete` to unblock.

## Auto-Detection

`bun PAI/TOOLS/ArchitectureSummaryGenerator.ts generate` detects threshold changes and writes stubs automatically. Thresholds:
- New subsystem row added to the Subsystem Reference table in CLAUDE.md
- New pipeline domain row added to the Pipeline Topology table
- Algorithm version bump in the Architecture Overview section
